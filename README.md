# Session Importer

Скрипты для импорта сессий из веб-интерфейса DeepSeek в структурированный репозиторий с поддержкой Obsidian.

## Структура репозитория

```
.
├── README.md
├── import-session.sh      # bash-скрипт для извлечения JSON из буфера обмена
├── import_session.py      # python-скрипт для парсинга и создания файлов
├── .session-index.json    # индекс сессий (created → uuid)
├── tmp/                   # временные файлы (сырые JSON)
├── sessions/
│   └── <session_uuid>/
│       ├── session.md     # метаданные сессии + оглавление
│       └── <turn_uuid>.md # отдельный файл для каждой реплики
└── tags/
    └── <tag_slug>/
        └── <turn_uuid>.md -> ../../sessions/.../turn.md
```

## Требования

- `bash`
- `python3`
- `jq` — для валидации JSON
- `xsel` (Linux) или `pbpaste` (macOS) — для работы с буфером обмена
- `pip install pyyaml` (опционально, если понадобится)

## Экспорт сессии из DeepSeek

В конце диалога (или в нужном срезе) запустите скрипт `dsi import`. Он скопирует
в клипбоард содержимое bin/prompts/import.txt. Вставьте содержимое в конец среза.

После получения ответа нажмите кнопку **"Copy"** в правом верхнем углу блока с JSON — всё содержимое скопируется в буфер обмена.

## Импорт

Запустите bash-скрипт:

```bash
./import-session.sh
```

Скрипт:
1. Извлечёт JSON из буфера обмена
2. Сохранит сырую копию в `tmp/`
3. Проверит валидность JSON и наличие обязательных полей
4. Создаст структуру `sessions/<uuid>/` и файлы реплик
5. Создаст символьные ссылки в `tags/<tag>/`

## Формат файлов

### `sessions/<uuid>/session.md`

```yaml
---
uuid: <session_uuid>
title: <заголовок>
summary: <краткое содержание>
created: <timestamp>
updated: <timestamp>
---

# <заголовок>

<краткое содержание>

## Обороты диалога

- [[<turn_uuid>|<вопрос (первые 60 символов)>]]
- ...
```

### `sessions/<uuid>/<turn_uuid>.md`

```yaml
---
uuid: <turn_uuid>
session: <session_uuid>
timestamp: <timestamp>
tags: [<tag1>, <tag2>]
---

**Вопрос:**

<текст вопроса>

**Ответ:**

<текст ответа>
```

### Символьные ссылки в `tags/`

```
tags/technology/<turn_uuid>.md -> ../../sessions/<session_uuid>/<turn_uuid>.md
```

Это позволяет просматривать все сообщения с определённым тегом как отдельные заметки в Obsidian.

## Obsidian

Репозиторий можно открыть как Obsidian-хранилище. Преимущества:

- **Граф связей** — каждый оборот связан с сессией
- **Поиск по тегам** — встроенный или через Dataview
- **Навигация** — по каталогам или через внутренние ссылки

### Пример Dataview-запроса

```dataview
TABLE timestamp, tags
FROM "sessions"
WHERE contains(tags, "linux")
SORT timestamp DESC
```

## Обработка дубликатов

Скрипт использует индекс `.session-index.json`, где хранится соответствие между `created` (timestamp первого сообщения) и `uuid` сессии. При повторном экспорте той же сессии (с тем же `created`) используется существующий UUID, а новые обороты добавляются в существующую папку.
